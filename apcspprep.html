WHAT IS THE CREATE PERFORMANCE TASK?

The Create Task is worth 30% of your AP CSP exam score. You write a program of your choice, record a short video of it running, and submit three written responses. The written responses are where most students lose points — not the code itself.

The College Board scores each response on a checklist. You either earn the point or you do not — there is no partial credit per row. The key is learning to write exactly what the rubric is looking for.

⚡ The three written prompts ask you to: (1) describe your program's purpose and what it does, (2) explain how your code uses abstraction, and (3) describe an algorithm you developed and how it works — including sequencing, selection, and iteration.
REQUIRED: State the overall purpose REQUIRED: Describe what the video shows TIP: Be specific — not just "it runs"

Imagine a student built a Playlist Mood Matcher — an app that takes your current mood as input and suggests a Spotify playlist genre.

❌ WEAK RESPONSE
"My program is a music app. In the video it shows the program running and the user clicking buttons to get music suggestions."
Missing: No specific purpose stated. "Shows the program running" tells the scorer nothing. No mention of input, output, or what problem it solves.
✅ STRONG RESPONSE
"The purpose of my program is to help users discover music that matches their current emotional state. The video shows a user entering 'stressed' as their mood input. The program processes this through a mood-to-genre mapping algorithm and displays 'lo-fi hip hop' as the recommended genre output."
Earns the point: Specific purpose stated. Input and output identified. What the video shows is clearly described with detail.
🛸 Formula: "The purpose of my program is to [solve specific problem]. The video shows [specific user action] which causes [specific program behavior] resulting in [specific output]."
REQUIRED: Name your procedure REQUIRED: Describe what it does REQUIRED: Explain how it manages complexity TIP: Show it being called with different inputs

Same student, Playlist Mood Matcher. They wrote a procedure called getMoodGenre(mood) that takes a mood string and returns a genre.

❌ WEAK RESPONSE
"I used a procedure called getMoodGenre. It uses a list to find the right genre. It makes the program easier to read."
Missing: Does not explain what the procedure actually does. "Makes the program easier to read" is vague — needs to explain HOW it manages complexity (reuse, hiding detail, shorter code elsewhere).
✅ STRONG RESPONSE
"My procedure getMoodGenre(mood) takes a mood string as a parameter and searches a list of mood-genre pairs to return the matching genre string. This procedure manages complexity because it is called three times in my program — once for the initial input, once after a quiz result, and once to refresh the suggestion. Without this abstraction, I would have repeated the same 15 lines of search code three times. The procedure hides those details and lets the rest of the program work at a higher level."
Earns the point: Names the procedure and parameter. Describes exactly what it does. Explains specifically how it manages complexity (reuse, hiding detail, avoiding repetition).
⚡ Key phrase: "This manages complexity because..." — then explain reuse, hiding implementation details, or reducing duplicate code. Concrete beats vague every time.
REQUIRED: Describe the algorithm REQUIRED: Include sequencing, selection, AND iteration REQUIRED: Explain how it works step by step TIP: The algorithm must be one YOU developed, not a built-in function

The student developed the mood-matching algorithm inside getMoodGenre — it loops through a list and uses a condition to find a match.

❌ WEAK RESPONSE
"My algorithm loops through a list of moods to find the matching genre. It uses iteration and selection to find the answer and return it."
Missing: Just names the concepts without explaining HOW they work together. No step-by-step description. Scorer cannot verify sequencing, selection, and iteration are all present.
✅ STRONG RESPONSE
"My algorithm inside getMoodGenre works as follows: First (sequencing) the procedure initializes a result variable to 'unknown' and sets an index to 1. Then (iteration) it uses REPEAT UNTIL to loop through each element of the moodList until the index exceeds the list length. Inside the loop (selection) an IF statement checks whether the mood at the current index matches the input parameter. If it matches, the algorithm sets result to the corresponding genre and sets a flag to stop the loop. Finally (sequencing) the procedure returns the result value."
Earns the point: Step-by-step. All three building blocks (sequencing, selection, iteration) explicitly identified and described. Scorer can clearly see how the algorithm functions.
✅ Checklist before submitting Prompt 3: Can you find the word "sequencing" (or describe ordered steps)? Can you find "selection" (IF/ELSE)? Can you find "iteration" (loop)? If any is missing, your response will not earn the point.
HOW THIS WORKS

The AP exam uses its own pseudocode notation from the official reference sheet. Read each program, trace through it step by step, then type what you think the output will be. The tracer checks your answer and explains why.

💻 Key syntax reminders: a ← expression assigns a value. DISPLAY(x) prints x followed by a space. List indexes start at 1. MOD returns the remainder.
